## 🗄️ Database Schema

### Users Table
```
users
├── id (auto-increment primary key)
├── user_id (TEXT, UNIQUE) - UUID generated by backend
├── name (TEXT) - User's full name
├── email (TEXT, UNIQUE) - Login email
├── password_hash (TEXT) - Bcrypt hashed password (NOT plain text!)
├── language (TEXT) - Preferred language for translation
├── online_status (BOOLEAN) - Currently online?
├── created_at (TIMESTAMP) - Account creation time
└── updated_at (TIMESTAMP) - Last update time

Indexes:
├── idx_users_email - Fast lookup by email (for login)
└── idx_users_user_id - Fast lookup by user_id
```

### Call History Table (Linked to Users)
```
call_history
├── id (auto-increment primary key)
├── call_id (TEXT, UNIQUE) - UUID for each call
├── caller_id (TEXT, FK → users.user_id) - Who called
├── callee_id (TEXT, FK → users.user_id) - Who was called
├── duration (INTEGER) - Call length in seconds
├── status (TEXT) - "completed", "missed", etc.
└── timestamp (TIMESTAMP) - When call happened

Indexes:
├── idx_call_history_caller - Fast lookup of user's calls
└── idx_call_history_callee - Fast lookup of calls received
```

### Chat Messages Table (Linked to Users)
```
chat_messages
├── id (auto-increment primary key)
├── message_id (TEXT, UNIQUE) - UUID for message
├── sender_id (TEXT, FK → users.user_id) - Who sent it
├── receiver_id (TEXT, FK → users.user_id) - Who received it
├── message (TEXT) - Message content
└── timestamp (TIMESTAMP) - When sent

Indexes:
├── idx_chat_sender - Fast lookup of sent messages
└── idx_chat_receiver - Fast lookup of received messages
```

## 🔐 Password Security Details

### Before Database Storage (Frontend)
1. User enters password: `SecurePass123!`
2. Validated client-side for:
   - Length ≥ 8 characters
   - Contains digit (0-9)
   - Contains special character (!@#$%^&*)

### During Transit (Frontend → Backend)
- Password sent in JSON POST body
- Should use HTTPS in production

### Backend Processing
```python
from bcrypt import hashpw, gensalt

# When user registers:
password = "SecurePass123!"
salt = bcrypt.gensalt()
password_hash = bcrypt.hashpw(password.encode(), salt).decode()
# Result: "$2b$12$jK7g9oC/3.D5wJ2QpP/8e...."

# Store password_hash in database (NOT the plain password)
```

### During Login
```python
from bcrypt import checkpw

# When user logs in:
stored_hash = "$2b$12$jK7g9oC/3.D5wJ2QpP/8e...."
entered_password = "SecurePass123!"

if checkpw(entered_password.encode(), stored_hash.encode()):
    print("Login successful!")
else:
    print("Invalid password!")
```

## 🎯 Key Points

1. **No Plain Text Passwords:** Passwords are hashed with bcrypt before storage
2. **UUID for Users:** Each user gets a unique UUID (not email)
3. **Email Unique:** Only one account per email address
4. **Secure Authentication:** Password verification uses bcrypt on backend
5. **Session Storage:** User info saved to localStorage when logged in
6. **Indexes for Speed:** Database indexes on email and user_id for fast lookups

## 🚀 Complete User Journey

```
1. User visits index.html
   ├─ "Get Started" → register.html (step 1)
   └─ "Login" → login.html

2. Registration Path:
   register.html (name, email, language)
   → password.html (password creation & validation)
   → Backend: /register (hash password, create UUID)
   → Supabase: INSERT into users table
   → Success: Auto-redirect to dashboard
   → Data saved: localStorage + Supabase database

3. Login Path:
   login.html (email, password)
   → Backend: /login (verify email, verify password)
   → Supabase: SELECT from users WHERE email=?
   → bcrypt check: password vs password_hash
   → Success: Auto-redirect to dashboard
   → Data saved: localStorage
   → Backend knows who you are: user_id is used for calls, messages, etc.

4. Application Usage:
   dashboard.html → Call users (Calls saved to call_history table)
   → Chat with users (Messages saved to chat_messages table)
   → View history (Retrieved from call_history table)
```
